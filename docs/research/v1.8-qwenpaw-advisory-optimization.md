# v1.8.1 QwenPaw Advisory Optimization Smoke

## Summary

v1.8.1 closes the v1.8 advisory optimization spike by fixing the QwenPaw
SSE typed-content parser and rerunning the real local QwenPaw matrix.

The observed root cause was not prompt failure. QwenPaw returned separate
reasoning/thinking content and final assistant message content over SSE. The
previous parser flattened both streams together, so the strict
`json_no_preamble_validator` saw reasoning text before the JSON object and
misclassified valid advisory output as `json_preamble_detected`.

After the parser fix, all three real local matrix cells returned
`advisory_qualified` with exact JSON output and zero repair attempts.

## Parser Fix

The parser now treats QwenPaw SSE as typed content instead of flat text:

- `object=message,type=reasoning` is excluded from advisory validation.
- Streaming `object=content,type=text` deltas are grouped by `msg_id`.
- Content is accepted only when the owning message is `type=message`.
- Completed response envelopes that include both reasoning and message output
  also skip reasoning message content.

This preserves strict JSON validation while avoiding false negatives caused by
agent reasoning streams.

## Boundary

- This smoke remains manual and guarded by `RUN_LIVE_QWENPAW_SMOKE=1`.
- It calls only a localhost QwenPaw service.
- It does not approve plans, publish notices, mutate merchant runtime, or
  create coupon/redemption records.
- `advisory_qualified` means the response passed the v1.8 required-field,
  validator, and authority-claim checks. It still does not prove production
  orchestration.
- The v1.4 fake QwenPaw shadow adapter remains the accepted product path.
- The v1.3 deterministic live demo remains the reliable demo path.
