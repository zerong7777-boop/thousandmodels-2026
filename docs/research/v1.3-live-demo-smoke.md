# v1.3 Live Demo Smoke

## Summary

v1.3 verifies that the v1.2 event-page and merchant-edge loop can run through a real local FastAPI backend and a real Vite browser session. The smoke is separate from the mocked Playwright visual suite.

## Commands

| Command | Working directory | Result |
| --- | --- | --- |
| `python scripts\reset_demo_state.py` | `apps/api` | exit 0; reset JSON printed for `demo-night-tour` and demo accounts |
| `python -m pytest tests/test_v12_event_page_merchant_edge.py -q` | `apps/api` | exit 0; 14 backend event-page and merchant-edge tests passed |
| `npm.cmd exec -- playwright test tests/e2e/v13-live-demo-smoke.spec.ts --config playwright.live.config.ts` | `apps/web` | exit 0; 1 live smoke passed and 5 screenshots generated |
| `rg -n "page\.route|route\(|page\.evaluate|fetch\(" apps\web\tests\e2e\v13-live-demo-smoke.spec.ts` | project root | exit 1; no route mocks, browser `fetch`, or `page.evaluate` fallback found in the v1.3 live smoke |

## Boundary Notes

- Public H5 remains unauthenticated.
- The live smoke uses the real backend API; it does not mock API responses.
- Public scan, claim, and redeem are exercised through the actual H5 UI controls.
- Public interaction controls expose only visitor-safe touchpoint/coupon fields for the current plan version.
- Qwen and DashScope are not required for this deterministic live demo smoke.
- QwenPaw workflow orchestration is not implemented in this phase.
- Merchant and public pages must not expose raw model/backend terms.
