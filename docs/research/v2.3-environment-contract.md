# v2.3 Environment Contract

Date: 2026-06-27

## Purpose

This document defines the environment variables required to run the current application in local, demo, staging, and production-like modes. It is an operations contract for the existing FastAPI, React/Vite, SQLite, and optional model-smoke tooling. It is not a cloud provisioning guide.

## Required Variables

| Area | Variables | Notes |
| --- | --- | --- |
| App mode | `APP_ENV`, `DEMO_MODE` | `APP_ENV` is one of `local`, `demo`, `staging`, `production`. Demo mode must be false in staging/production. |
| API security | `APP_SECRET_KEY`, `ALLOWED_ORIGINS`, `CSRF_MODE`, `SESSION_COOKIE_SECURE`, `SESSION_SAMESITE`, `SESSION_TTL_HOURS` | Staging/production require a non-empty secret, explicit HTTPS origins, secure cookies, and double-submit CSRF. |
| Store | `DATABASE_URL`, `AUTO_MIGRATE` | Local/demo may auto-migrate SQLite. Staging/production use explicit migration steps with `AUTO_MIGRATE=false`. |
| Agent | `AGENT_BACKEND`, `AGENT_DRAFT_BACKEND`, `DASHSCOPE_API_KEY`, `QWEN_MODEL` | Deterministic agents are the default. Qwen remains optional and non-authoritative. |
| QwenPaw smoke | `RUN_LIVE_QWENPAW_SMOKE`, `QWENPAW_BASE_URL`, `QWENPAW_AGENT_ID` | Manual localhost smoke only. It must not be enabled in staging/production startup. |
| Web | `VITE_API_BASE_URL`, `VITE_DEMO_MODE` | Web demo affordances should match API demo mode. |

## Mode Matrix

| Variable | Local | Demo | Staging | Production-like |
| --- | --- | --- | --- | --- |
| `APP_ENV` | `local` | `demo` | `staging` | `production` |
| `DEMO_MODE` | `true` | `true` | `false` | `false` |
| `APP_SECRET_KEY` | optional dev fallback | optional dev fallback | required secret value | required secret value |
| `CSRF_MODE` | `demo` | `demo` | `double-submit` | `double-submit` |
| `SESSION_COOKIE_SECURE` | `auto` or `false` | `auto` or `false` | `true` or `auto` | `true` or `auto` |
| `ALLOWED_ORIGINS` | localhost origins | demo HTTPS or localhost during rehearsal | explicit HTTPS origins | explicit HTTPS origins |
| `DATABASE_URL` | local SQLite path | demo SQLite path | staging database URL | production database URL |
| `AUTO_MIGRATE` | `true` | `true` | `false` | `false` |
| `AGENT_BACKEND` | `deterministic` | `deterministic` | `deterministic` unless explicitly reviewed | `deterministic` unless explicitly reviewed |
| `AGENT_DRAFT_BACKEND` | `deterministic` or guarded `qwen` | `deterministic` or guarded `qwen` | `deterministic` by default | `deterministic` by default |
| `RUN_LIVE_QWENPAW_SMOKE` | manual `true` only for smoke | manual `true` only for smoke | `false` | `false` |
| `VITE_DEMO_MODE` | `true` | `true` | `false` | `false` |

## Startup Policy

Staging and production-like startup must fail before serving traffic when any of these are true:

- `DEMO_MODE=true`.
- `APP_SECRET_KEY` is missing.
- `SESSION_COOKIE_SECURE=false`.
- `CSRF_MODE` is not `double-submit`.
- `ALLOWED_ORIGINS` is empty, non-HTTPS, or malformed.
- `AUTO_MIGRATE=true`.
- `RUN_LIVE_QWENPAW_SMOKE=true`.
- `QWENPAW_BASE_URL` is an explicit localhost URL outside local/demo.

## Deployment Check Commands

Run local API checks:

```powershell
cd apps/api
python -m pytest -q
```

Run the deployment smoke against a running API:

```powershell
python apps\api\scripts\deployment_smoke.py --base-url http://127.0.0.1:8000
```

Run the explicit migration command before staging/production-like startup:

```powershell
python apps\api\scripts\migrate_store.py
```

## Boundaries

This contract does not add managed secrets, TLS, DNS, CDN, object storage, managed database backups, incident operations, or real external integrations. Qwen and QwenPaw remain optional advisory tooling and cannot approve, publish, or mutate authoritative state.
