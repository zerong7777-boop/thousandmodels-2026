# v1.3 Operator Demo Script

## Goal

Show that Zhiyin Haojiang is a multi-role cultural-tourism event operations Agent demo, not a static page. The demo runs through real local FastAPI state and a real browser session.

## Pre-Demo Reset

Run from `apps/api`:

```powershell
python scripts\reset_demo_state.py
```

Then start the normal local API and web app, or run the live Playwright smoke command if generating evidence.

## 3-5 Minute Flow

| Time | Route | Role | Action | What to say |
| --- | --- | --- | --- | --- |
| 0:00-0:30 | `/login` | Organizer | Sign in as organizer. | The system is role-separated and starts from real demo auth. |
| 0:30-1:10 | `/organizer/events/demo-night-tour` | Organizer | Show published event page, plan evidence, and merchant edge package readiness. | The organizer controls plan approval, event publication, and merchant preparation. |
| 1:10-1:45 | `/merchant/dashboard` | Merchant | Sign in as merchant and show package, touchpoints, and coupon rules. | The merchant sees only their operational package, not organizer internals. |
| 1:45-2:15 | `/merchant/events/demo-night-tour/status` | Merchant | Report sold out. | Merchant runtime state feeds the central operations loop. |
| 2:15-2:50 | `/organizer/events/demo-night-tour/exceptions` | Organizer | Show incident and generate operation suggestion. | The system turns merchant signals into reviewable operations evidence. |
| 2:50-3:30 | `/public/events/demo-night-tour` | Public | Scan, claim, and redeem. | The visitor page is public, mobile-facing, and avoids backend/model terms. |
| 3:30-4:20 | `/review/demo-night-tour` | Organizer | Show touchpoint summary and merchant outcomes. | The event produces measurable follow-up evidence, not just generated text. |
| 4:20-5:00 | docs evidence | Presenter | Show v1.3 smoke and optional Qwen result. | Qwen is optional and controlled; deterministic demo remains reliable without it. |

## Allowed Claims

- The v1.3 demo runs locally through FastAPI and Vite.
- Organizer, merchant, and public surfaces are separate.
- Merchant runtime updates can trigger organizer-side operational review.
- Public interactions feed review metrics.
- Qwen is optional controlled draft evidence when enabled.

## Claims Not To Make

- QwenPaw orchestrates the workflow.
- Qwen autonomously approves, publishes, mutates inventory, creates coupons, or creates route versions.
- The demo depends on DashScope.
- The project is connected to real merchants, POS, payment, real map APIs, real weather APIs, or real traffic prediction.
