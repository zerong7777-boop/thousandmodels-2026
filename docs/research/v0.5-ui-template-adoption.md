# v0.5 UI Template Adoption

Date: 2026-06-11  
Project: Zhiyin Haojiang  
Spec: `docs/proposal/v0.5-ui-polish-spec.md`

## Decision

Primary direction:

`shadcn/ui blocks + satnaing/shadcn-admin visual density + owned Tailwind product components`.

Use official shadcn/ui block patterns for dashboard, login, sidebar, metric, card, badge, and panel composition. Use `satnaing/shadcn-admin` as a page-density and shell-composition reference, not as a copied app. Add product-specific local components under `apps/web/src/components/product/` rather than adding a new UI framework.

Backup direction:

`TailAdmin React` as a dashboard density reference only if shadcn/satnaing patterns are insufficient for organizer pages. Do not copy TailAdmin source in the primary path.

Adoption mode:

- Copy/adapt small open-source component patterns only when license permits.
- Prefer reference-only use for commercial product pages and proprietary screenshots.
- Do not import a full admin application.
- Do not add a new major UI dependency in v0.5 Task 1.
- Implementation must not start until this document is reviewed.

## Implementation Rules

- Copy small local component patterns only when license permits it.
- Prefer adaptation into owned files over adding new UI dependencies.
- Do not import a full admin application.
- Do not add a new major UI framework.
- Do not use template shell changes as a substitute for page workflow changes.
- Visitor-facing pages may use proprietary product references as inspiration only, not copied UI.
- The v0.5 implementation should create product components under `apps/web/src/components/product/` and keep them small.
- The accepted path must keep `DASHSCOPE_API_KEY`, Qwen, and QwenPaw out of the required demo flow.
