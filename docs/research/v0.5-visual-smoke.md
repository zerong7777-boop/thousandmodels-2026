## Design Review Rubric

| Dimension | Score rule |
| --- | --- |
| Product credibility | 1 = wireframe/demo, 5 = credible product page |
| Role differentiation | 1 = same skin, 5 = role-specific experience |
| Information density | 1 = sparse, 5 = dense but readable |
| Visual hierarchy | 1 = unclear action/status, 5 = status and next action obvious |
| Mobile ergonomics | 1 = cramped/overflow, 5 = touch-friendly and stable |
| Cultural/event atmosphere | 1 = backend data, 5 = visitor-facing cultural route |
| Low backend leakage | 1 = internal terms visible, 5 = product language only |

## High-Impact Page Changes

| Page | Visible v0.5 change | Why it passes |
| --- | --- | --- |
| Login | Product access split layout with role-specific demo quick-fill. | It no longer reads as a plain identity switcher; credential login remains primary. |
| Organizer dashboard | Operations header, needs-attention queue, KPI strip, and activity timeline. | The organizer can see status, risk, and next action from the first viewport. |
| Exception center | Queue, impact scope, recovery confirmation, before/after diff, and public notice preview. | The page now expresses a human decision workflow rather than a backend object list. |
| Merchant status | Large touch-first quick actions appear before the advanced report form. | The merchant can report operational state without filling a form first. |
| Public H5 | Visitor route header, live update, route progress, story stops, and visitor tasks. | The public page is now a visitor-facing cultural route instead of a backend preview. |
| Organizer review | KPI-backed review report with recommendations. | The review center is metric-backed, not only prose. |

## Remaining Weaknesses

| Weakness | Follow-up |
| --- | --- |
| The organizer visual style is now credible but still conservative. | A later polish pass can add stronger Macau old-district visual identity after content direction is finalized. |

| Public and tourist route pages use text-only story cards. | Add licensed or generated cultural imagery only after the route concept and content rights are clearer. |
| Review metrics are deterministic demo values. | Keep them deterministic for the required demo path; later Qwen/QwenPaw work can improve wording, not state mutation. |
