# v2.6 Browser Release Evidence Gate Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a browser release evidence gate that proves the deterministic local beta works through the real frontend and backend.

**Status:** Implemented. Verification is recorded in `docs/ai/VERIFY.md`.

**Architecture:** Extend the existing live Playwright path instead of creating a separate runner. Keep the gate as one targeted Playwright spec with real FastAPI and Vite web servers, API setup through real HTTP calls, UI assertions through browser routes, local screenshot capture, and committed JSON evidence. Use env-driven ports and a default isolated SQLite database in the live config.

**Tech Stack:** Playwright, Vite, FastAPI, SQLite, TypeScript, Node standard library.

---

## Task 1: Live Playwright Config Hardening

**Files:**

- Modify: `apps/web/playwright.live.config.ts`
- Modify: `apps/web/playwright.config.ts`
- Modify: `apps/web/package.json`

- [x] **Step 1: Add env-driven live ports**

Update `playwright.live.config.ts` so:

- `PLAYWRIGHT_API_PORT` controls the API port;
- `PLAYWRIGHT_WEB_PORT` controls the Vite port;
- `VITE_API_BASE_URL` and `PLAYWRIGHT_BASE_URL` can override the derived URLs;
- the test process and Vite process share the same API base URL.

- [x] **Step 2: Add isolated SQLite default**

Set `DATABASE_URL` for the live API web server to an isolated SQLite file under the OS temp directory when the caller has not supplied one.

- [x] **Step 3: Keep ordinary Playwright runs non-live**

Exclude `v26-browser-release-evidence.spec.ts` from `apps/web/playwright.config.ts`, matching the existing v1.3 live smoke exclusion.

- [x] **Step 4: Add a package script**

Add:

```json

```
