**Goal:** Turn the v1.7 real QwenPaw advisory smoke failure into a measured v1.8 optimization spike with prompt variants, validator variants, bounded repair, and sanitized evidence that distinguishes provider reachability from advisory qualification.

## 2. Implementation decisions

This plan resolves the open decisions in the spec as follows:

1. v1.8 writes a new evidence path under `docs/research/assets/v1.8-qwenpaw-advisory-optimization/` so it does not overwrite the v1.5/v1.7 historical artifact.
2. Prompt variants are implemented as a small in-script registry. This avoids a new dependency or external prompt file for the spike.
3. Repair is implemented in `live_qwenpaw_smoke.py` as a bounded second or third QwenPaw call. It defaults to zero repair attempts unless `QWENPAW_MAX_REPAIR_ATTEMPTS` is set.
4. `json_no_preamble_validator` becomes the preferred v1.8 validator for JSON-only prompt variants. Existing Markdown parsing remains available through `strict_existing_validator` for regression compatibility.
5. The script supports agent variants through environment selection. It does not create or mutate local QwenPaw agent configuration.

def _parse_exact_advisory_json(text: str) -> tuple[dict[str, str], bool]:
    stripped = text.strip()
    if not stripped.startswith("{") or not stripped.endswith("}"):
        return {}, False
    try:
        payload = json.loads(stripped)
    except json.JSONDecodeError:
        return {}, False
    if not isinstance(payload, dict):
        return {}, False
    fields = {
        field: _extract_json_text(payload.get(field)).strip()
        for field in REQUIRED_ADVISORY_FIELDS
        if field in payload
    }
    return fields, True
