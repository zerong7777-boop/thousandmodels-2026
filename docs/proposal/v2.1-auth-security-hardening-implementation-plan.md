# v2.1 Auth, Session, And Demo Boundary Hardening Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Separate demo authentication behavior from staging/production security behavior while preserving the local deterministic demo.

**Architecture:** Add a small settings layer in the API, route all auth/session/CSRF decisions through it, and expose a CSRF endpoint for non-demo modes. The frontend gets a mutation helper that uses the demo header only in demo mode and fetches signed CSRF tokens otherwise.

**Tech Stack:** FastAPI, Python standard library, Pydantic models already in the repo, React/Vite, Vitest, pytest.

```python
SETTINGS = load_settings()

@app.on_event("startup")
def validate_and_seed_on_startup() -> None:
    SETTINGS.validate_startup()
    if SETTINGS.demo_mode:
        seed_demo_accounts(STORE)
```

## Task 4: CSRF Double-Submit Baseline

**Files:**

- Create: `apps/api/app/csrf.py`

- Modify: `apps/api/app/main.py`
- Test: `apps/api/tests/test_v21_csrf_policy.py`

- [ ] **Step 1: Add CSRF tests**

Cover demo acceptance, production rejection of `demo`, valid double-submit acceptance, and invalid token rejection.

- [ ] **Step 2: Implement token signing**

Create helpers:

```python
def issue_csrf_token(secret: str, now: datetime | None = None) -> str: ...
def verify_csrf_token(token: str, secret: str, max_age_seconds: int = 7200) -> bool: ...
```

Use HMAC SHA-256 with timestamp and random nonce. Do not store server-side state in v2.1.

- [ ] **Step 3: Add endpoint**

Add:

```python
@app.get("/api/auth/csrf")
def csrf(response: Response):
    token = issue_csrf_token(SETTINGS.required_secret())
    response.set_cookie("zhiyin_csrf", token, httponly=False, secure=SETTINGS.session_cookie_secure, samesite=SETTINGS.session_samesite, path="/")
    return {"csrf_token": token}
```

- [ ] **Step 4: Update `verify_mutation_origin()`**

Use settings:

- demo mode accepts existing local origins and `demo`;
- non-demo mode requires allowed origin plus matching cookie/header token.
