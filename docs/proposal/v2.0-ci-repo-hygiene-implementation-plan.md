# v2.0 CI And Repository Hygiene Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add enforceable CI and repository hygiene gates for the existing MVP without changing product behavior.

**Architecture:** GitHub Actions runs three independent jobs: API tests, web tests/build, and repository hygiene. A dependency-free Python script owns local and CI hygiene checks so the logic is versioned and reproducible.

**Tech Stack:** GitHub Actions, Python 3.11, Node 20, pytest, npm/Vitest/Vite, Python standard library.

---

## File Structure

| Path | Action | Responsibility |
| --- | --- | --- |
| `.github/workflows/ci.yml` | create | CI entrypoint for API, web, and hygiene jobs. |

| `docs/ai/NEXT.md` | modify | Move next phase to v2.1 auth/security hardening after CI is pushed and observed. |

## Task 1: Add Repo Hygiene Script

**Files:**

- Create: `scripts/repo_hygiene.py`

- [ ] **Step 1: Create the script**

Implement a Python standard-library CLI with:

- repeated `--check` values,
- optional `--base`,
- tracked text-file scanning,
- changed-file generated PNG checking,
- clear pass/fail output,
- non-zero exit on failure.

- [ ] **Step 2: Run focused script checks**

Run:

```powershell
python scripts\repo_hygiene.py --check secrets
python scripts\repo_hygiene.py --check local-paths
python scripts\repo_hygiene.py --check node-modules
python scripts\repo_hygiene.py --check generated-artifacts --base origin/main
```

Expected:

- All commands exit `0`.
- The generated-artifacts command does not fail on the unstaged historical PNG files because it checks committed changes relative to `origin/main`.
