> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Turn the existing v0.4 role-separated skeleton into a visually credible, screenshot-backed v0.5 product demo without changing the deterministic backend workflow.

**Architecture:** Keep the current FastAPI, SQLite, React, Vite, TypeScript, Tailwind, and local shadcn-style primitives. Treat Playwright screenshots and source/text gates as first-class acceptance evidence.

**Tech Stack:** FastAPI, Pydantic, SQLite, pytest, React 19, Vite, TypeScript, Vitest, Testing Library, Playwright, Tailwind CSS, local shadcn-style UI primitives, lucide-react, existing Ant Design only as a migration bridge.

## 0. Scope Lock

This plan implements `docs/proposal/v0.5-ui-polish-spec.md`.

Hard constraints:

- Do not change the backend deterministic orchestration loop unless a frontend integration bug is discovered and documented.
- Do not require Qwen, QwenPaw, `DASHSCOPE_API_KEY`, model training, or fine-tuning.
- Do not connect real merchants, hardware, payments, POS systems, traffic prediction, or real map APIs.
- Do not implement public registration or a full SaaS tenant/user-management system.
- Do not add a marketing landing page.
- Do not import a full admin template as the application.
- Do not start source implementation before Task 1 is complete and reviewed.
- Do not accept visual changes that only change color, spacing, or shell navigation while page workflows remain unchanged.
- Do not expose internal backend terms to tourist or public pages.
- Do not use Ant Design `Layout`, `Sider`, or `Menu` in accepted product shells.
