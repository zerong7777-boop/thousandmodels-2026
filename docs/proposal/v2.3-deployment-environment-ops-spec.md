# v2.3 Deployment And Environment Operations Spec

Date: 2026-06-27

Status: proposed.

## Purpose

v2.3 defines how the application is configured, started, and smoke-tested across local, demo, staging, and production-like environments. Earlier phases make the repo testable, secure demo boundaries explicit, and persistence migration-aware. v2.3 turns those controls into repeatable operational commands.

This phase is not a full cloud launch. It prepares the project for a controlled staging deployment.

## Readiness Target

v2.3 moves the project from "local app with CI" to "staging-deployable app with explicit environment contracts".

## In Scope

- Define environment variable contract for API and web.
- Add startup checks for environment, auth, database, migrations, and optional provider features.
- Add readiness endpoint distinct from health.
- Add deployment smoke script.
- Add Dockerfiles only if they remain minimal and reflect existing local commands.
- Update `.env.example` and docs with local/staging/production differences.
- Ensure Qwen/QwenPaw live features remain manual and disabled by default outside local/demo.

## Out Of Scope

- Real cloud provisioning.
- Domain, TLS, DNS, CDN, object storage, or managed database setup.
- Secret manager integration.
- Horizontal scaling.
- Kubernetes.
- Payment, POS, map, traffic, weather, or identity integrations.

## Required Behavior

### Environment Contract

Document required variables:

| Area | Variables |
| --- | --- |
| App mode | `APP_ENV`, `DEMO_MODE` |
| API security | `APP_SECRET_KEY`, `ALLOWED_ORIGINS`, `CSRF_MODE`, `SESSION_COOKIE_SECURE`, `SESSION_SAMESITE` |
| Store | `DATABASE_URL`, `AUTO_MIGRATE` |
| Agent | `AGENT_BACKEND`, `AGENT_DRAFT_BACKEND`, `DASHSCOPE_API_KEY`, `QWEN_MODEL` |
| QwenPaw smoke | `RUN_LIVE_QWENPAW_SMOKE`, `QWENPAW_BASE_URL`, `QWENPAW_AGENT_ID` |
| Web | `VITE_API_BASE_URL`, `VITE_DEMO_MODE` |

### Startup Checks

API startup must validate:

- auth settings from v2.1;
- schema migration state from v2.2;
- allowed origins are explicit outside local/demo;
- live QwenPaw smoke flags are not accidentally enabled in production;
- demo reset/seed behavior is disabled outside demo mode.

### Health And Readiness

`/api/health` remains lightweight.

Add `/api/ready` with:

- settings mode;
- store connectivity;
- migration pending count;
- auth policy mode;
- optional provider status summarized as enabled/disabled, not raw secrets.

Readiness must not expose local absolute paths or secrets.

### Deployment Smoke

Add a script that can run against a base URL:

```text
GET /api/health
GET /api/ready
GET /api/public/events/demo-night-tour
```

In demo mode it may optionally log in as organizer and check `/api/auth/me`.

## Tests

Tests must cover:

- readiness endpoint redacts sensitive values;
- production-like config refuses unsafe QwenPaw live flags;
- deployment smoke handles success and failure responses;
- `.env.example` includes all documented keys.

## Acceptance Criteria

- API tests pass.
- Web tests and build pass.
- Deployment smoke passes against a locally running API.
- CI remains green.
- Documentation explains how to run local, staging-like, and production-like checks.

## Commercial Readiness Impact

v2.3 partially closes v1.9 `CR-P0-005`: deployment and environment policy. It does not close real hosting, managed secrets, TLS, monitoring, backups, or incident operations.
