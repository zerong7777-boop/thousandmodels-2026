# v2.5 Live E2E Release Gate Spec

Date: 2026-06-28

Status: implemented.

## Purpose

v2.5 adds a release gate that proves the current local beta can run through the core product state machine against a real running API. Earlier phases added CI, auth boundaries, migrations, readiness, observability, and runbooks. This phase turns those pieces into one pre-release command that produces bounded evidence.

The gate is API-first. It complements the v1.3 live Playwright demo smoke and v2.3 deployment smoke without replacing either one.

## Readiness Target

v2.5 moves the project from "operable beta candidate" to "release-gated local beta candidate".

## In Scope

- A script that runs against a live API base URL.
- Health and readiness checks.
- Request ID and error-envelope sanity checks.
- Demo organizer, merchant, and tourist/public path checks using real HTTP sessions.
- Demo seed, plan generation, plan approval, event-page draft/publish, and merchant-edge package generation.
- Public H5 projection check through `GET /api/public/events/{event_id}`.
- Public touchpoint scan, coupon claim, and redemption simulation through public APIs.
- Merchant workbench check for the generated interaction package.
- Review report generation check.
- Metrics endpoint check for process-local counters touched during the gate.
- JSONL step output and optional JSON evidence artifact.
- Unit tests for success, failure, redaction, and evidence shape.

## Out Of Scope

- Starting or supervising FastAPI or Vite.
- Browser screenshots or Playwright visual evidence.
- Real cloud deployment.
- Vendor monitoring, alerting, SLOs, or incident routing.
- Real Qwen, DashScope, or QwenPaw calls.
- Real merchant/POS/payment/map/weather/traffic integrations.
- Production data reset.

## Required Behavior

### Invocation

The release gate should be runnable as:

```powershell
python apps\api\scripts\release_gate.py --base-url http://127.0.0.1:8000
```

Optional arguments:

- `--event-id`, default `demo-night-tour`;
- `--origin`, default `http://127.0.0.1:5173`;
- `--output`, optional JSON evidence path.

### Gate Steps

The script must perform these steps in order:

1. `GET /api/health`, require HTTP 200 and `status=ok`.
2. `GET /api/ready`, require HTTP 200 and `status=ready`.
3. `GET /api/events`, require HTTP 401 and the v2.4 error envelope before login.
4. Organizer login with `organizer.demo` / `demo1234`.
5. `GET /api/auth/me`, require organizer role.
6. Seed demo event.
7. Generate plan.
8. Approve plan version 1.
9. Draft and publish event page.
10. Generate merchant-edge packages.
11. Public event projection includes an event page and no raw internal terms.
12. Run public scan, coupon claim, and redemption against generated touchpoint/coupon IDs.
13. Merchant login with `merchant.m001.demo` / `demo1234`.
14. Merchant workbench exposes the generated interaction package.
15. Organizer login again.
16. Generate review report.
17. `GET /api/metrics`, require touched counters to be present.

### Evidence

Each step should emit one JSON line:

```json
{"step":"health","ok":true,"status_code":200,"summary":"ok"}
```

If `--output` is provided, write a JSON object containing:

- `status`: `passed` or `failed`;
- `base_url`;
- `event_id`;
- `started_at`;
- `finished_at`;
- `duration_ms`;
- `steps`;
- `metrics_observed`;
- `failure`, when failed.

Evidence must not include cookies, session tokens, CSRF tokens, Authorization headers, raw provider payloads, or local absolute paths.

### Failure Semantics

The command exits `0` only when all required steps pass.

It exits non-zero when:

- the API is unreachable;
- a required endpoint returns the wrong status;
- a required payload field is missing or unsafe;
- a public surface leaks forbidden internal terms;
- metrics expected from the gate are missing.

### Boundary

The gate proves deterministic local beta release readiness only. It does not prove cloud readiness, production monitoring, real external integrations, real customer traffic, or production QwenPaw orchestration.

## Tests

Tests must cover:

- successful full gate execution using fake request responses;
- failure when a required endpoint is not ready;
- failure when public projection leaks internal terms;
- failure when metrics are missing;
- JSONL and evidence output are bounded and redacted;
- CLI argument defaults.

## Acceptance Criteria

- v2.5 spec and implementation plan exist.
- Release gate script exists under `apps/api/scripts/`.
- Focused release gate tests pass.
- Full backend pytest passes.
- Web tests and build pass.
- Repository hygiene and whitespace checks pass.
- A real local API run of the release gate passes and writes evidence outside tracked generated screenshot paths.
- `docs/ai/STATUS.md`, `docs/ai/VERIFY.md`, and `docs/ai/NEXT.md` record the result and boundary.
